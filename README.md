# 蜀道数据采集系统管理后台

这是一个基于 Vue 3 和 Element Plus 构建的现代化数据采集插件管理后台系统，专注于提供直观易用的数据采集管理体验。

## 功能特性

### 🎯 已完成功能

-   ✅ 现代化的响应式界面设计
-   ✅ 完整的顶部导航栏（品牌信息、菜单导航、通知中心、用户信息）
-   ✅ 采集插件管理页面（核心功能）
    -   四大数据统计卡片（活跃插件、今日采集量、文档数量、采集状态）
    -   完整的插件列表展示（搜索、状态筛选、分页、批量操作）
    -   插件状态图表（饼图显示运行状态分布）
    -   采集性能图表（趋势分析和实时监控）
    -   最近采集数据展示（数据流实时跟踪）
    -   文档管理卡片（快速访问和统计）
-   ✅ 专业的字体系统（方正公文小标宋 + 完善的备用字体方案）
-   ✅ 圆角风格设计语言
-   ✅ 完全响应式布局（桌面端、平板、移动端）

### 🚧 开发中功能

-   🔄 文档管理模块（界面已就绪，功能开发中）
-   🔄 系统设置模块（界面已就绪，功能开发中）

## 技术栈

-   **前端框架**: Vue 3 (Composition API)
-   **UI 框架**: Element Plus 2.3.12
-   **图标库**: @element-plus/icons-vue 2.1.0
-   **图表库**: ECharts 5.6.0 + Vue-ECharts 6.6.1
-   **路由管理**: Vue Router 4.2.4
-   **构建工具**: Vite 4.4.9
-   **开发语言**: JavaScript

## 项目架构

### 🏗️ 组件化页面结构

采用页面组件分离的架构设计，每个页面都有独立的文件夹和专用组件：

```
src/
├── components/                    # 全局共享组件
│   ├── TopNavbar.vue             # 顶部导航栏（完整功能）
│   ├── Layout.vue                # 主布局组件
│   └── AppFooter.vue             # 页脚组件
│
├── views/                        # 页面模块
│   ├── PluginsPage/              # 插件管理页面 ✅ 已完成
│   │   ├── index.vue             # 页面主文件
│   │   └── components/           # 页面专用组件
│   │       ├── StatsCard.vue     # 统计卡片组件
│   │       ├── PluginsList.vue   # 插件列表（搜索+表格）
│   │       ├── PluginStatusChart.vue # 状态饼图
│   │       ├── PerformanceChart.vue  # 性能图表
│   │       ├── RecentCollectionData.vue # 最近采集数据
│   │       └── DocumentCard.vue  # 文档管理卡片
│   ├── AnalyticsPage/            # 数据分析页面 🔄 开发中
│   │   ├── index.vue             # Coming Soon 界面
│   │   └── components/           # 页面专用组件（待开发）
│   ├── DocumentsPage/            # 文档管理页面 🔄 开发中
│   │   ├── index.vue             # Coming Soon 界面
│   │   └── components/           # 页面专用组件（待开发）
│   └── SettingsPage/             # 系统设置页面 🔄 开发中
│       ├── index.vue             # Coming Soon 界面
│       └── components/           # 页面专用组件（待开发）
├── utils/                        # 工具函数
│   └── fontUtils.js              # 字体管理工具
├── router/                       # 路由配置
│   └── index.js                  # 路由定义
├── style.css                     # 全局样式
├── main.js                       # 应用入口
└── App.vue                       # 根组件
```

### 📁 设计原则

-   **页面独立**: 每个功能页面拥有独立的文件夹和组件生态
-   **组件分离**: 页面特定组件与全局共享组件清晰分离
-   **渐进开发**: 支持功能模块的渐进式开发和扩展
-   **维护友好**: 清晰的文件组织便于团队协作和长期维护

## 安装与运行

### 环境要求

-   Node.js 16.0+
-   npm 或 yarn

### 1. 安装依赖

```bash
npm install
```

### 2. 启动开发服务器

```bash
npm run dev
```

### 3. 构建生产版本

```bash
npm run build
```

### 4. 预览生产版本

```bash
npm run preview
```

### 5. 字体配置（可选）

系统使用方正公文小标宋作为主字体，详细安装指南请参考 [FONT_INSTALL_GUIDE.md](./FONT_INSTALL_GUIDE.md)

## 核心功能模块

### 🔌 插件管理（主功能）

#### 数据概览仪表盘

-   **活跃插件数**: 实时统计当前运行的插件数量
-   **今日采集量**: 展示当日数据采集总量和增长趋势
-   **文档总数**: 统计已采集和管理的文档数量
-   **系统状态**: 整体系统运行状态和健康度指标

#### 插件列表管理

-   **全功能搜索**: 支持插件名称的实时搜索
-   **状态筛选**: 按运行状态（运行中/已停止/异常）筛选
-   **详细信息**: 插件头像、名称、描述、版本信息
-   **批量操作**: 支持多选和批量启动/停止操作
-   **快速操作**: 单个插件的启动、停止、配置操作

#### 可视化监控

-   **状态分布图**: 饼图展示插件运行状态分布
-   **性能趋势图**: 采集性能的时间序列分析
-   **实时数据流**: 最近采集数据的实时展示
-   **文档统计**: 文档分类和存储状态概览

### 📊 其他功能模块（开发中）

-   **文档管理**: 计划包含文档分类、预览、导出、搜索功能
-   **数据分析**: 计划包含趋势分析、性能统计、报表生成
-   **系统设置**: 计划包含参数配置、权限管理、系统维护

## 开发规范

### 代码标准

-   使用 Vue 3 Composition API
-   遵循 Element Plus 设计规范
-   组件化开发思路
-   响应式设计优先

### 组件开发规范

-   全局组件放置在 `src/components/`
-   页面组件放置在 `src/views/PageName/components/`
-   每个页面的组件只在该页面内使用
-   保持组件的单一职责原则

### 样式规范

-   使用 scoped 样式防止污染
-   统一的圆角设计（20px）
-   响应式断点：480px, 768px, 1200px
-   主色调：#4285f4, #34a853, #fbbc04
